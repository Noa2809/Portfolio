Le Projet

Le projet TARDIS analyse les données de retard des trains SNCF pour prédire les délais grâce au Machine Learning.

L'architecture repose sur trois notebooks couvrant le pipeline complet :

  • tardis_eda.ipynb : Nettoyage des données et Data Visualization (EDA).
  • tardis_model.ipynb : Entraînement et comparaison des modèles d'Intelligence Artificielle.
  • tardis_dashboard.py : Dashboard interactif développé avec Streamlit.

Analyse Exploratoire (EDA)

Étape 1

Data Cleaning

Suppression des colonnes inutiles, extraction des mois/années. Passage de données brutes à un dataset nettoyé.

Étape 2

Distribution des retards

Majorité des trains à l'heure, mais présence d'une longue traîne positive (retards importants ponctuels).

Étape 3

Saisonnalité

Pic de retards très marqué en juillet (8.4 min) et creux en mai, lié aux départs en vacances.

Étape 4

Top trajets impactés

L'axe Italie - Paris Lyon arrive en tête avec plus de 14 minutes de retard moyen.

Résultats Machine Learning

Nous avons comparé 3 modèles d'apprentissage automatique pour prédire les retards :

Modèle 1 (Vainqueur)

Random Forest

MAE : 2.01 min
Score R² : 43%

Modèle 2

XGBoost

MAE : 2.26 min
Score R² : 34%

Modèle 3 (Baseline)

Linear Regression

MAE : 2.53 min
Score R² : 24%
