Objectif du projet

OWASP Juice Shop est une application web volontairement truffée de failles de sécurité, conçue pour apprendre le pentest dans des conditions réalistes.

Notre mission a été d'explorer chaque fonctionnalité (compte, login, panier, administration), d'identifier les vulnérabilités et de les documenter avec des recommandations correctives.

Résultat : 58 challenges résolus sur 100, touchant à plus de 5 catégories critiques de l'OWASP.

Principales vulnérabilités exploitées

SQL Injection

Contournement de la page de connexion pour accéder à des comptes sans mot de passe.

Broken Access Control

Accès non autorisé au panneau d'administration via la manipulation des requêtes.

XSS (Cross-Site Scripting)

Injection de scripts malveillants stockés via la barre de recherche et les avis.

Sensitive Data Exposure

Fuite de données utilisateurs confidentielles directement via les réponses de l'API REST.

Ce que j'ai appris

  • Offensive Security : Compréhension et exploitation des attaques web courantes.
  • Outils : Utilisation de Burp Suite pour intercepter et modifier le trafic HTTP.
  • Analyse : Rétro-ingénierie du code source frontend et observation des routes API.
  • Documentation : Rédaction de 58 rapports de vulnérabilité (Writeups) structurés en Markdown.
